# ShushTLS v0.5.0

Agent-friendly API surface, richer certificate metadata, and documentation aimed at automation (Kubernetes, cert-manager, constrained bootstrap).

## API

### OpenAPI and interactive docs

- Machine-readable spec at **`GET /api/openapi.json`** (OpenAPI 3.1).
- Interactive explorer at **`GET /api/docs`**.
- Human-oriented guide at **`GET /docs`** (links to the spec and covers profiles, ACME, and trust bootstrap).

ACME endpoints are documented via `externalDocs` on the OpenAPI root (directory at `/acme/directory`).

### Errors

API errors use **RFC 9457** `application/problem+json` (`title`, `status`, `detail`). Wrong HTTP methods on known paths return **405**; unknown `/api/*` paths return **404**.

### Certificate issuance

- **`POST /api/certificates`** accepts optional **`common_name`** and subject fields per request, without changing global leaf-subject defaults (e.g. CN `node` with SAN `db.internal` for CockroachDB/etcd-style integrations).
- **`POST /api/certificates`** with **`Content-Type: application/pkcs10`** (or PEM CSR body) signs a CSR with the standard leaf profile and returns the signed certificate. The private key stays on the client; ShushTLS stores the cert only.

### Certificate list

- **`GET /api/certificates`** returns enriched metadata: common name, serial, SHA-256 fingerprint, key/signature algorithms, key usage, extended key usage, and related fields.
- **`GET /api/certificates?fields=basic`** returns the previous minimal list shape for callers that only need SANs and validity.

The **`is_service`** flag on list entries identifies ShushTLS’s own HTTPS certificate.

### Bundle downloads

**`GET /api/certificates/{san}`** bundle downloads (tar/zip) now include:

- **`ca.pem`** — root CA
- **`chain.pem`** — leaf + root

Query parameters:

| Parameter | Values | Effect |
|-----------|--------|--------|
| `names` | `san` (default), `generic` | `generic` uses `cert.pem` / `key.pem` instead of SAN-prefixed filenames |
| `format` | `k8s-tls` | Emits a ready-to-apply Kubernetes `Secret` (`kubernetes.io/tls`) as YAML |
| `type` | `tar`, `zip` | Archive format (unchanged) |

## Documentation (web UI)

- **`/docs`** — Certificate profile (algorithms, KU/EKU, validity, subject construction, wildcard SAN expansion).
- **`/docs`** — ACME endpoint reference; constrained-environment CA trust (bootstrap fetch, fingerprint, cert-manager/Docker/k8s examples).
- **`/acme`** — RFC 8555 directory “What’s supported” table (including intentional omissions: `revokeCert`, `keyChange`, `newAuthz`).
- **`/acme`** — cert-manager integration notes (ACME trap, self-check, CronJob example).
- **`/trust`** (Install CA) — Root CA SHA-256 fingerprint for out-of-band verification.

## Upgrade notes

- **API clients:** Parse `application/problem+json` for errors instead of legacy ad-hoc JSON bodies.
- **List consumers:** Default list responses are enriched; use `?fields=basic` if you depend on the older field set.
